When Character Movement SPD Meets Condition

Triggers when "Monitor Movement Rate" unit status effect meets the condition.

Output Values

  • Event Source Entity (Entity) - Character entity that has the "Monitor Movement Rate" unit status effect and met its condition.
  • Event Source GUID (GUID) - Always 0.
  • Unit Status Config ID (Config ID) - Configuration ID of the unit status. e.g. 1012345678.
  • Condition: Comparison Type (Enumeration) - One of Less Than, Less Than or Equal To, Greater Than, Greater Than or Equal To. It cannot be Equal To.
  • Condition: Comparison Value (Floating Point Numbers) - Speed to compare with, as specified in the "Monitor Movement Rate" unit status effect. Unit is meter/second.
  • Current Movement SPD (Floating Point Numbers) - Current speed of the character entity. Unit is meter/second.

Usage

To use this node, first a unit status must be created with an effect "Monitor Movement Rate". In the example below, this effect will trigger when the character's speed is greater than 5.00 meter/second, and it will not trigger again for 1.50 seconds while in cooldown. Next, character must have the unit status so that it can monitor the speed. You can also use Add Unit Status to add the unit status to the character.

A character can have multiple "Monitor Movement Rate" unit status effects, so Unit Status Config ID can be used to tell them apart with Multiple Branches.

You may want to use Query Character's Current Movement SPD instead if you just need to get the speed, but you still need the unit status effect on the character.

Notes

  • Event Source Entity and Event Source GUID may not be the same as the self entity, if it was forwarded by Forwarding Event.
  • Even if "Trigger CD" is set to 0.00 or negative value, this node only triggers about 2 times per second. So the lowest effective "Trigger CD" is roughly 0.5 seconds. There is a huge delay between character accelerating and this node triggering, and that may have to do with very infrequent event polling rate, but we cannot be sure.
  • Current Movement SPD includes the vertical component. When falling or jumping, the character may have higher speed than expected.
  • The walking speed of Manekin/Manekina is roughly 5. Sprinting speed of Manekin/Manekina is roughly 8, but it can reach around 10 briefly.

Performance

This node took ~7 units to run on average.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
