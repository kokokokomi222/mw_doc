Tech

Because this is a simple static site, I just manually wrote everything in raw HTML, CSS, and Javascript. There is only ~200 lines of Javascript code, which handles search, documentation loading, and showing bigger images when clicked.

The documentation HTML is generated from "markdown". I wrote a simple markdown parser in Python for MW Doc markdown variant. Other build tasks like templating the HTML files, generating placeholder documentation, and doing grammar/spell checks are also written in Python.

No LLM was used to write any part of the code (I do use it to look things up). I think LLM is not helpful even for this kind of tiny prototype weekend projects. The quality is just not there.

I currently do not use any tracking on this site. I don't even know how many visits I get.
