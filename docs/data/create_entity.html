Create Entity

Creates a pre-placed entity.

Input Parameters

  • Target GUID (GUID) - GUID for the entity to create. Typically copied from the top of the entity details panel. e.g. 1012345678
  • Unit Tag Index List (Integer List) - List of unit tag IDs to put on the created entity. e.g. [1023456789, 1023456790]. Can be left empty.

Usage

Typically, entities placed in the scene are created when the stage begins. However, "Initial Create" can be set to false under "Create Settings" in the entity detail panel, which allows you to create these entities later with this node.

This node can also be used to recreate entities that have been removed or destroyed, but it cannot be used for entities created by Create Prefab or Create Prefab Group because these entities do not have GUID.

Use Create Prefab instead if you want to be able to dynamically set the created entity's location, rotation, level, and even the prefab. This is the most common way to create an entity.

Unit tags let you find all entities with a specific tag. See Add Unit Tag to Entity for more info. It can be left empty. See "Notes" section below for more detail.

Consider using Activate/Disable Entity Deployment Group instead if you want to create many entities at once.

Notes

  • If you set unit tags in the entity detail panel, those tags will be overwritten by Unit Tag Index List. For example, if tag A and B were set in the entity detail panel, but tag B and C were passed into Unit Tag Index List, then the created entity has tag B and C. However, if Unit Tag Index List is left empty, or an empty list is passed into it, the tags set in the entity detail panel will be used.
  • If the entity with Target GUID has been created already, this node is no-op. In particular, this node does not modify the tag list of the entity in this case.
  • If the entity with Target GUID does not exist, this node raises a "GUID does not exist" error, but continues the execution.
  • An entity removed with Remove Entity or Destroy Entity can be created again with this node.
  • This node works perfectly fine with creations.
  • There is a limit of 1000 entities on the field. If this node is used when the limit is already reached, this node raises "GUID setting error" error (misleading because it should be an "Exceeded the maximum number of entities" error instead), but continues execution.

Performance

Creating an entity with the default dynamic unit setting took ~65 units to run on average. Setting Unit Tag Index List does not seem to visibly affect the performance of this node.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
