Query Character's Current Movement SPD

Gets the speed and the velocity of a character.

Input Parameters

  • Character Entity (Entity) - Character to get the speed from

Output Values

  • Current Speed (Floating Point Numbers) - Speed of the character. Unit is meter / second.
  • Velocity Vector (3D Vector) - Velocity of the character. Unit of the components of the vector is meter / second.

Usage

To use this node, first a unit status must be created with an effect "Monitor Movement Rate". The parameters of this unit status is not relevant for this node: any values will work. Those parameters are for When Character Movement SPD Meets Condition. Next, character must have the unit status so that it can monitor the speed. You can also use Add Unit Status to add the unit status to the character. This node functions even if the condition of the effect is not satisfied, as long as the effect is on the character.

Notes

  • If called in When Character Movement SPD Meets Condition, Current Speed of this node is always equal to Current Movement SPD of the event node.
  • Square of Current Speed should be approximately equal to the sum of squares of the components of Velocity Vector.
  • The walking speed of Manekin/Manekina is roughly 5. Sprinting speed of Manekin/Manekina is roughly 8, but it can reach around 10 briefly.
  • The character speed is measured very inconsistently on a moving platform. Even while the platform is moving at constant speed without turning, the speed seems to be sometimes relative to the platform, and sometimes it seems relative to the world. This is probably because the server doesn't know the precise local position of the character immediately.

Performance

This node took ~3 units to run on average.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
