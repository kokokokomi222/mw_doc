Switch Current Interface Layout

Switch Interface Layout.

Input Parameters

  • Target Player (Entity) - Entity of the player to switch interface layout for.
  • Layout Index (Integer) - ID of the interface layout. e.g. 1073741825

Usage

Interface layout is a collection of UI controls. Interface layouts can be created in the "Manage UI Control Groups" panel, which can be opened from the main menu. Each player has one interface layout displayed. The initial layout can be configured in the class detail panel. This node lets you switch to a different layout for a particular player.

Example

In this example, a new interface layout named Koko's Layout was created with ID 1073741845. The node graph changes the interface layout of the player of the character that selected the tab.

Notes

  • Layout Index can be selected from the list of interface layouts by clicking the magnifying glass icon.
  • If Target Player is not a player entity, this node raises "Entity does not exist" error (can be misleading, because the entity may exist) and continues the execution.
  • If Layout Index is not an ID for an interface layout, this node is no-op.
  • Some UI controls play subtle animations when switched in by this node. Thus, if you switch from layout A to layout B and then back to layout A in one node graph execution, it can be noticed by the players by the entering animations playing.

Performance

  • This node took ~6 units to run on average.
Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
