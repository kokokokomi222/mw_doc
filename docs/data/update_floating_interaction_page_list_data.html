Update Floating Interaction List Data

This documentation was published soon after the release of this node. Some of the information may be inaccurate. Let me know if you find something wrong.

Updates the list of items of tab or single-choice window.

Input Parameters

  • Player Entity (Entity) - Entity of the player to update the page for.
  • List Index (Integer) - ID of the "Tab" or "Single-Choice Window" UI control to update. e.g. 1073741825
  • Visible List Item (List) - IDs of the items in the UI control. List of integers. e.g. [1, 2, 3]
  • Select First Item by Default (Boolean) - Yes to select the first item. No otherwise.

Usage

Read Show Floating Interaction Page to learn how to use floating interaction page. This node lets you decide which items of "Tab" or "Single-Choice Window" to show, and in what order.

Example

In this example, we created a tab UI control in a floating interaction page. The tab UI control has three items named Ayaka, Barbara, and Columbina, and their preset texts are set accordingly. In the node graph, we update the content of the tab to [3, 1] when a specific UI control is triggered. As a result, only Columbina tab and Ayaka tab are visible, with Columbina tab selected.

Notes

  • If Player Entity is not a player entity, this node raises "Entity does not exist" error (can be misleading because the entity may exist), but continues the execution.
  • If List Index is not an ID for a UI control with list items, this node is no-op.
  • If Visible List Item contains a number that is not the ID of a list item, it is ignored. For example, if a tab has ID 1, 2, and 3, then [0, 2, 4] was passed into Visible List Item, the result of this node is that only the list item with ID 2 is visible, because 0 and 4 are not IDs and are ignored.
  • As seen in the "Example" section above, the list items are laid out in the order given in Visible List Item.
  • If Select First Item by Default is No, it retains the previous selection if the selected item is visible after this node. If it is not visible, it will select the first item in Visible List Item. For example, let's say three list items [1, 2, 3] are initially visible, and Visible List Item is [3, 1] and Select First Item by Default is No.
    • If 1 was selected, 1 is selected after this node runs (it retains the previous selection).
    • If 2 was selected, 3 is selected after this node runs (it selects the first item in the new list).
    You can use this behavior to select a specific item as demonstrated in the example below.
  • If "Return to Server Event" is turned on in the list item panel, When Floating Interaction Page is Triggered triggers on this UI control if this item gets selected after this node executes even if the selection did not change.

Performance

This node took ~6 units to run on average.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
