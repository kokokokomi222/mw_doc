Show Floating Interaction Page

This documentation was published soon after the release of this node. Some of the information may be inaccurate. Let me know if you find something wrong.

Opens floating interaction page.

Input Parameters

  • Player Entity (Entity) - Entity of the player to open the floating interaction page for.
  • Floating Interaction Page Index (Integer) - ID of the floating interaction page to open. e.g. 1073741825
  • Initialize List Data (Dictionary that maps integer to list of Integers) - Dictionary where keys are IDs of "Tab" or "Single-Choice Window" UI controls (e.g. 1073741825) and values are lists of visible item IDs of the control (e.g. [1, 2, 3]). Can be left blank.

Usage

Luna VI introduced various new UI controls, but floating interaction page is the most complex one. It allows craftspeople to create a beautiful UI page that works with mouse, touch screen, and controller. It is often overlaid and takes up full screen.

Floating interaction page cannot be added directly to a layout. Instead, this node is used to open the page defined in the "UI Control Group Library", similar to how deck selector works. See "Example" section below to see how a floating interaction page can be used.

Item lists for tabs and single-choice windows can be updated dynamically with Update Floating Interaction List Data.

Example

We will start by configuring a floating interaction page in "Manage UI Control Groups". First, we must ensure that the UI Rendering mode is "Hierarchical Rendering" instead of "Sequential Rendering". Some of the new UI controls are only supported under the new "Hierarchical Rendering" mode. Stages created after the start of Luna VI should default to "Hierarchical Rendering", however older stages may need to adjust this mode.

Next, we create a floating interaction page in "UI Control Group Library". Contents of the page can be edited under "Style" > "Edit Details". After filling out the contents, save the template and copy the ID of the template.

Lastly, we create a node graph to open the page for the player that select the tab. We also make item 1, 2, and 3 visible for the tab UI control with ID 1073741962.

Notes

  • If Player Entity is not a player entity, this node raises "Entity does not exist" error (can be misleading because the entity may exist), but continues the execution.
  • If Floating Interaction Page Index is not an ID for a floating interaction page, this node is no-op.
  • If the floating interaction page with the same ID is already open, this node is no-op.
  • If a floating interaction page with different ID is already open, this node opens the specified floating interaction page. As a result, two pages will be open, one on top of the other. Only the page with the higher layer value will be interactable (the most recently opened page in case of layer value tie). When the interactable page is closed, the other page will now be interactable.
  • If Initialize List Data is empty, every item of tabs and single-choice windows will not be visible.
  • If a key of Initialize List Data is not an ID for a UI control with list items, it ignores that entry. Other legitimate entries will be processed.
  • If an entry of a value of Initialize List Data is not a valid ID for a list item, it ignores that entry. Other legitimate entries will be processed.
  • Floating interaction pages do not retain selection state when closed. When they are re-opened with this node, all tabs and single-choice windows are completely refreshed.

Performance

This node took ~7 units to run on average.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
