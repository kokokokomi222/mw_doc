Bitwise AND

Performs bitwise AND.

Input Parameters

  • Value 1 (Integer) - First operand.
  • Value 2 (Integer) - Second operand.

Output Values

  • Result (Integer) - Result of bitwise AND of the two input values.

Usage

In general programming, bitwise operations are most often used in low level programming or when encoding/decoding binary data. For Miliastra, a potential application is for compression of data. If you have 10000 boolean states in your stage, it may be more feasible and performant to have one list of 500 integer 20-bit-fields, instead of using a list of 10 structures of a boolean list of length 1000.

Example

In this example, we are performing bitwise AND of 26 and 19. Clearly, 26 is 11010 in binary. 19 is 10011 in binary. So only the 2nd bit (representing 2^1) and the 5th bit (representing 2^4) is 1 in both numbers. Thus, the output is 10010 in binary, which is 18 in decimal. 

    11010 (26 in decimal)
AND 10011 (19 in decimal)
---------
    10010 (18 in decimal)

Notes

  • For negative input, Miliastra uses two's complement to represent the value in binary. For example, 5 AND -1 is 5 because -1's binary representation is 11111111111111111111111111111111 (32 1's) in 32-bit integer with two's complement. We recommend that you avoid using negative integers in bitwise operations.

Performance

This node took ~1 unit to run on average.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
