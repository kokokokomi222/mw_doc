When Custom Variable Changes

Triggers when a custom variable changes its value.

Output Values

  • Event Source Entity (Entity) - Entity that holds the changed custom variable.
  • Event Source GUID (GUID) - GUID of the above entity. 0 if the entity was created dynamically.
  • Variable Name (String) - Name of the changed custom variable.
  • Pre-Change Value (Generic) - The custom variable's old value before the change.
  • Post-Change Value (Generic) - The custom variable's new value after the change.

Usage

Use this node to listen to event where a custom variable changes. Trigger Event must be set to Yes on Set Custom Variable for this node to work.

If you want to listen to changes to multiple custom variables, you will want to use Multiple Branches where Variable Name is connected to Control Expression. Even when you only listen to change of one custom variable, it may be a good practice to use Multiple Branches for future maintenance of the node graph. This node only triggers if Pre-Change Value and Post-Change Value are set to the same type as the custom variable. Place this node for each type of custom variables you want to listen to. See "Notes" section below for more detail.

Even though the official documentation states that this node does not work for "Vessel-type Custom Variables", it works perfectly fine for container type variables as long as Set Custom Variable was used to change the value.

This node could be used to organize node graphs, sort of treating the node as functions of a program, by grouping logic that may be called multiple times. Such "function" can be called from other entity's node graph as well. However, these "functions" do not run on change right away in the middle of the node graph execution. Instead, it goes into the event queue and runs later. A signal is probably better to use if multiple node graphs may want to listen to it, or if it passes multiple parameters. But if you just want to pass in one parameter to one node graph, this method may be easier to reason with, because it is limited to just the node graphs held by the entity. Also, there is a limit of 99 signals per stage.

Notes

  • If the previous value and the new value is the same, it still triggers this event.
  • Trigger Event must be set to Yes on Set Custom Variable to trigger this event. If a custom variable is originally set to A, a Set Custom Variable with Trigger Event set to No changes the value to B, and then another with Trigger Event set to Yes changes the value to C, only one event triggers this node with Pre-Change Value B (not A) and Post-Change Value C.
  • Event Source Entity and Event Source GUID may not be the same as the self entity, if it was forwarded by Forwarding Event.
  • Changing the type of Pre-Change Value or Post-Change Value will change the type of the other to match it.
  • If the type of the Pre-Change Value and Post-Change Value are not selected and left as generic, verification fails with "Node parameter settings error". Note that this behavior differs from When Node Graph Variable Changes.
  • If the type of the Pre-Change Value and Post-Change Value does not match the type of the custom variable that changed, it does not trigger the node. If you want to listen to changes to multiple custom variables with differing types, place this node for each type.
  • If the type of the Pre-Change Value and Post-Change Value is a dictionary, it will trigger for change of any dictionary custom variable, but it will only return Pre-Change Value and Post-Change Value if the key and value types match. For dictionaries with mismatching key/value type, it will return an empty dictionary for both outputs. We recommend that you do not use this node for dictionaries. If you really want to listen to changes to multiple custom variables with differing dictionary types, place this node for each dictionary type and use Multiple Branches to filter out custom variables with other dictionary types.
  • This node works for lists and dictionaries, but only if Set Custom Variable was used to set the entire value. Modifying a custom variable list with nodes like Set List Value or a custom variable dictionary with nodes like Set or Add Key Value Pairs to Dictionary does not trigger this node.
  • This node also works for structures, providing the correct Pre-Change Value and Post-Change Value. Like list and dictionary type, this node only triggers when Set Custom Variable is used.
  • If a value of the custom variable was changed multiple times during an execution of a node graph, each change triggers a separate event. All events are polled in order after the execution of the node graph.
  • If Set Custom Variable creates a new custom variable that did not exist, then this node successfully triggers with Pre-Change Value set to the default value of the type.

Performance

Listening to the change of an integer variable took ~3 units to run on average.

Listening to the change of a list of 1000 integers took ~15 units to run on average.

Set Custom Variable runs slightly faster if Trigger Event is set to No. See Set Custom Variable for detail.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
