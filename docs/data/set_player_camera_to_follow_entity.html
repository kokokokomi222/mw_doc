Set Player Camera to Follow Entity

This server node is only available in Beyond Mode.

This documentation was published soon after the release of this node. Some of the information may be inaccurate. Let me know if you find something wrong.

Switch camera to follow entity.

Input Parameters

  • Player Entity (Entity) - Entity of player to switch the camera for.
  • Follow Entity (Entity) - Entity to follow with the camera. Must have an "Object Camera" component.
  • Camera Template Name (String) - Name of the object camera, as specified in the "Object Camera" component.

Usage

Starting in Luna VI, craftsperson can configure the camera to follow an entity. This allows finer control over the position of the camera and its movement. Use this node to start following an entity with the camera. See "Example" section below to see how to configure an object camera.

Example

First, create an object camera from the "Manage Main Camera" panel. For this example, we name it Koko's Cinematic Camera. Next, create an "Object Camera" component on the entity that you want camera to follow. For this example, we selected the camera we created above and then named this object camera Koko's Cinematic Camera for Ball. Lastly, a node graph is created to switch the camera to the object camera.

Notes

  • If Player Entity is not a player entity, this node raises "Entity does not exist" error (can be misleading, because the entity may exist) but continues the execution.
  • If Follow Entity does not exist, this node is no-op.
  • If Follow Entity does not have an "Object Camera" component or Camera Template Name is not a name of an object camera specified in the component, this node is no-op.

Performance

This node took ~8 units to run on average.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
