Remove Entity

Removes an entity.

Input Parameters

  • Target Entity (Entity) - Entity to remove.

Usage

The difference between "remove" and "destroy" is important to understand. Firstly, "destroy" could be thought of as a more specific version of "remove". "Remove" and "destroy" both trigger When Entity Is Removed/Destroyed, but only "destroy" triggers When Entity Is Destroyed. When a creation is "destroyed", it plays its death animation and drops energy particles/orbs. However, if it is "removed", it simply disappears into thin air.

Notes

  • If the removed entity was statically created and thus have a GUID, then this entity can be recreated using Create Entity.
  • If Target Entity was set as an invalid entity 0, this node is no-op.
  • If Target Entity is the stage entity, this node is no-op. Stage cannot be removed.
  • If Target Entity is a player entity, this node is no-op. Player cannot be removed by this node, but it is removed if the player leaves the stage.
  • If Target Entity is a character entity, this node is no-op. Character cannot be removed by this node, but it is removed if the owning player leaves the stage. However, it can be destroyed by Destroy Entity.

Performance

To remove object entities with default setting, it took ~24 units to run on average. Whether the stage is listening to When Entity Is Removed/Destroyed does not seem to visibly affect the performance of this node.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
