When Equipment is Purchased

This documentation was published soon after the release of this node. Some of the information may be inaccurate. Let me know if you find something wrong.

Triggers when an inventory owner buys equipment.

Input Parameters

  • Purchasing Inventory Owner Entity (Entity) - Entity that bought the equipment.
  • Purchasing Inventory Owner GUID (GUID) - GUID of the above entity. 0 if the entity was created dynamically.
  • Equipment Index List (List) - IDs of the equipments bought. It is a list of integers. e.g. [2, 3, 5]

Usage

This node listens to the event where equipment purchase occurs between inventory owners via a shop. The shop owner entity must be running a "Self Inventory Shop", not "Custom Items Shop". The item purchased must be equipment, not a normal item. This node triggers in both situations:

  • A character buys from a self inventory shop, triggering this node on the character entity.
  • A character sells to a self inventory shop, triggering this node on the shop owner entity.

When Equipment is Sold is always triggered at the same time, but on the other entity of the transaction.

Equipment Index List is the list of equipment IDs. They are not equipment configuration IDs. Note that "equipment ID" is an integer that identifies an instance of equipment, and it is typically small (e.g. 3), while "equipment configuration ID" is a configuration ID that identifies a class of equipment, and it is typically large (e.g. 1107296257).

Performance

This node took ~10 units to run on average.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
