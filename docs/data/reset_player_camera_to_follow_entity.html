Reset Player Camera to Follow Entity

This server node is only available in Beyond Mode.

This documentation was published soon after the release of this node. Some of the information may be inaccurate. Let me know if you find something wrong.

Switches camera to follow player.

Input Parameters

  • Player Entity (Entity) - Entity of the player to switch the camera for.

Usage

Starting in Luna VI, craftsperson can configure the camera to follow an entity. This allows finer control over the position of the camera and its movement. After using Set Player Camera to Follow Entity, this node can be used to come back to the player camera.

In this example, we reset the camera to the player camera for the player that selects the tab.

Notes

  • If Player Entity is not a player entity, this node raises "Entity does not exist" error (can be misleading, because the entity may exist) but continues the execution.
  • If you switch from the default camera A to a player camera B and then to an object camera C, using this node will switch the camera back to camera B, not camera A.

Performance

This node took ~8 units to run on average.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
