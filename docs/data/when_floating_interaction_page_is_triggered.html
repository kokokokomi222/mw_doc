When Floating Interaction Page is Triggered

This documentation was published soon after the release of this node. Some of the information may be inaccurate. Let me know if you find something wrong.

Triggers when a UI control on a floating interaction page is used.

Output Values

  • Player Entity (Entity) - Entity of the player that triggered the interaction.
  • Player GUID (GUID) - GUID of the above player.
  • Floating Interaction Page Index (Integer) - ID of the floating interaction page that contains that used UI control. e.g. 1073741889
  • Interactive Item Index (Integer) - ID of the used UI control. e.g. 1073741890
  • List Index (List of Integers) - List with the ID of the used UI control as its only entry, if the UI control is a tab or a single-choice window, empty list otherwise. e.g. [1073741890] TODO: I'm not sure why this is a list instead of integer.
  • Selected List Item (List of Integers) - List with the ID of the selected item as its only entry, if the UI control is a tab or a single-choice window, empty list otherwise. e.g. [3] TODO: I'm not sure why this is a list instead of integer.

Usage

See Show Floating Interaction Page to learn how to open a floating interaction page. This node can be used to listen to interactions from the page, such as button press or switch toggle. Note that When UI Control Group Is Triggered do not trigger for UI controls on a floating interaction page. This node should be mounted on a player entity, but the event can be forwarded.

Notes

  • This node is triggered by the following UI controls:
    • Interactive Button
    • Item Display
    • Custom Button
    • Custom Switch
    • Single-Choice Window (if "Return to Server Event" is enabled)
    • Tab (if "Return to Server Event" is enabled)
    They do not trigger When UI Control Group Is Triggered if they are on a floating interaction page.

Performance

This node took ~8 units to run on average.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
