Search List and Return Value ID

Searches a list for a value, returning the list of indices.

Input Parameters

  • Target List (Generic) - The list to search in.
  • Value (Generic) - The value to search for.

Output Values

  • ID List (Integer List) - List of indices where the value is found at.

Usage

Use this node to find all occurrences of a value in a list. If you only need to know whether the value is present, use List Includes This Value instead.

Example

In this example, we have a string list depicting a result of a 10-pull. We search this list for occurrences of "Kokomi" and print the indices, so this example will print "2" and "7".

Notes

  • Changing the type of List or Value will change the type of the other to match it.
  • If no matches are found, ID List is empty.
  • For floating point numbers, this node uses approximate equality (e.g. 10.0 and 10.00001 is considered equal by this node). This approximate equality seems to be same as Equal, but we cannot be completely sure.

Performance

For an integer list of length 10, is took ~2 units to run on average.

For an integer list of length 1000, it took ~10 units to run on average with 0 matches, ~18 units to run with all entries matching.

Performance may vary for various reasons. These numbers are just for giving a sense of rough magnitude. Performance may differ from profiling in a test play when playing a published stage. First execution of a node within a node graph execution tend to take a bit longer.
